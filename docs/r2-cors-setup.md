# R2 CORS Configuration

This document explains how to apply CORS policy to your Cloudflare R2 bucket.

## CORS Policy Definition

The [r2-cors-policy.json](r2-cors-policy.json) file contains the CORS configuration needed for R2 to allow cross-origin requests from browsers and mobile apps.

### Policy Details:
- **AllowedOrigins**: `*` (allows requests from any origin)
- **AllowedMethods**: GET, PUT, POST, DELETE, HEAD
- **AllowedHeaders**: `*` (allows any header)
- **ExposeHeaders**: ETag, x-amz-version-id
- **MaxAgeSeconds**: 3600 (1 hour cache)

## How to Apply CORS to R2

### Option 1: Via Cloudflare Dashboard (Easiest)

1. Go to [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. Select your account → **R2**
3. Click on your bucket name
4. Go to **Settings** tab
5. Scroll to **CORS** section
6. Click **Add CORS rule**
7. Configure:
   - **Origin**: `*` (or specific domain: `https://yourdomain.com`)
   - **Methods**: Check GET, PUT, POST, DELETE, HEAD
   - **Headers**: `*` (or specify: Content-Type, Authorization, x-amz-*)
   - **Expose headers**: ETag, x-amz-version-id
   - **Max age**: 3600
8. Click **Save**

### Option 2: Via Wrangler CLI

```bash
# Install wrangler if you don't have it
npm install -g @cloudflare/wrangler

# Set CORS policy (requires authentication)
wrangler r2 bucket cors put <BUCKET_NAME> < r2-cors-policy.json
```

### Option 3: Via S3-Compatible API (AWS CLI)

```bash
aws s3api put-bucket-cors \
  --bucket <BUCKET_NAME> \
  --cors-configuration file://r2-cors-policy.json \
  --endpoint-url https://<ACCOUNT_ID>.r2.cloudflarestorage.com \
  --region auto
```

## Security Notes

For **production**:
- Replace `*` in **AllowedOrigins** with your specific domain: `https://yourdomain.com`
- Replace `*` in **AllowedHeaders** with specific headers: `Content-Type, Authorization`
- This prevents unauthorized cross-origin access

For **development/testing**:
- Keep `*` to allow requests from any origin (current setting)

## Verification

After applying CORS, test with:

```bash
curl -i -X OPTIONS https://<BUCKET_URL>/test.txt \
  -H "Origin: http://localhost:3000" \
  -H "Access-Control-Request-Method: GET"
```

Look for response headers like:
```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, PUT, POST, DELETE, HEAD
```

## For Android App

Your Android app's pre-signed URLs (generated by `/api/media/presign`) will work because:
1. Pre-signed URLs bypass CORS checks (no preflight needed)
2. The app uses JWT bearer token authentication
3. R2 validates the signature regardless of origin

CORS is mainly needed for:
- Web browser direct uploads/downloads
- Cross-origin fetch requests
